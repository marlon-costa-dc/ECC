# Uncloud Cluster Management

## When to Activate

Use when working with Uncloud clusters to:

- Bootstrap or join machines with `uc machine`
- Deploy services from Compose files with `uc deploy`
- Publish HTTP, HTTPS, TCP, or UDP ports
- Configure Caddy ingress with `x-caddy`, `x-ports`, or `--caddyfile`
- Route external LAN devices through the cluster proxy
- Inspect logs, service state, volumes, DNS, or machine placement

## Core Concepts

- **No central control plane** — equal peers connected by WireGuard
- **Caddy** runs globally on every machine and auto-obtains TLS
- **Overlay network** — services communicate via `10.210.0.0/16` by default
- **Caddyfile is autogenerated** — never edit it directly

## Essential CLI Commands

| Command | Purpose |
|---------|---------|
| `uc machine init user@host` | Bootstrap first machine |
| `uc deploy` | Deploy from `compose.yaml` |
| `uc scale SERVICE N` | Set replica count |
| `uc service logs SERVICE` | View logs |
| `uc service exec SERVICE` | Shell into container |
| `uc caddy config` | Show generated Caddyfile |

## Compose Extensions

Uncloud adds `x-ports`, `x-caddy`, and `x-machines` on top of Docker Compose:

```yaml
services:
  app:
    image: app:latest
    x-ports:
      - example.com:8000/https
    x-machines: [machine-1, machine-2]
  db:
    image: postgres:18
    volumes: [db-data:/var/lib/postgresql/data]
    x-machines: db-machine
volumes:
  db-data:
```
