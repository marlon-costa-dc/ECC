# brand-voice

## Quando usar
- The user wants content or outreach in a specific, consistent voice.
- Writing for X, LinkedIn, email, launch posts, threads, or product updates.
- Adapting a known author's tone across multiple channels.
- The content lane needs a reusable style system, not one-off mimicry.

## O que fazer
1. Gather 5–20 real samples in priority order: recent X posts/threads, articles/essays/memos/launch notes/newsletters, outbound emails/DMs that worked, product docs/changelogs/README/site copy.
2. Prefer recent material unless older work is more canonical.
3. Separate "public launch voice" from "private working voice" if sources split.
4. Extract operational patterns: rhythm, compression vs explanation, capitalization, parentheticals, question purpose, claim sharpness, numbers/mechanisms/receipts, transitions, forbidden tropes.
5. Produce a reusable `VOICE PROFILE` block using `references/voice-profile-schema.md`.
6. Reuse the profile in the same session; save to memory/workspace only if the user asks for a durable artifact.

## Regras críticas
- Use only real source material. Never use generic exemplars or invented styles.
- Delete and rewrite: fake curiosity hooks; "not X, just Y"; "no fluff"; forced lowercase; LinkedIn thought-leader cadence; bait questions; "Excited to share"; generic founder-journey filler; corny parentheticals.
- Keep the profile short enough to drop into session context.
- Do not create repo-tracked voice fingerprints unless explicitly asked.
- If another skill has a partial voice capture, this skill is the SSOT.

## Exemplo
```
VOICE PROFILE: Affaan / ECC
- direct, compressed, concrete
- specifics/mechanisms/receipts/numbers beat adjectives
- parentheticals qualify, never perform
- conventional capitalization unless intentional
- rare questions, never bait
- sharp, blunt, skeptical, or dry tone
- earned transitions
- never: fake hooks, "no fluff", "Excited to share", bait questions
```
